# 🤝 Contributing Guide

Thank you for your interest in contributing to this project! 🚀
To maintain consistency and quality, please follow the guidelines below.

---

## ⭐ Step 1: Support the Project

Before contributing, please **star this repository** to show your support.

---

## 🍴 Step 2: Fork the Repository

* Fork this repository to your own account.
* Clone your fork to your local machine:

```bash
git clone https://github.com/your-username/your-repo-name.git
cd your-repo-name
```

---

## ⚙️ Step 3: Setup Locally

* Install dependencies
* Run the project locally to ensure everything works fine

```bash
npm install
npm run dev
```

---

## 🐛 Step 4: Raise an Issue

* While using the application, if you find a bug or have a feature request:

  * Go to the **Issues** section
  * Clearly describe the problem or suggestion
  * Wait for approval or discussion before starting work

---

## 🌿 Step 5: Create a Contribution Branch

> 🚨 Always use a separate branch for contributions

```bash
git checkout -b contribution/your-feature-name
```

---

## 💻 Step 6: Make Changes

* Write clean and readable code
* Follow existing project structure and conventions
* Test your changes before submitting

---

## 🔄 Step 7: Commit Changes

```bash
git add .
git commit -m "feat: describe your changes clearly"
```

---

## 🚀 Step 8: Push and Create PR

```bash
git push origin contribution/your-feature-name
```

* Go to your fork on GitHub
* Click **"Compare & Pull Request"**
* Submit your PR with a clear description

---

## 📌 Contribution Rules

* Always create PRs from the `contribution/*` branch
* Do not push directly to `main`
* Keep PRs small and focused
* Link related issues in your PR

---

## 🙌 Final Note

Your contributions help improve this project for everyone.
Happy coding! 💻🔥
